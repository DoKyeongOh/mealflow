# 1차 목표 정리

## 목표
회원 관리와 기본 식료품 관리 기능을 중심으로,  
사용자가 자신의 식료품 상태와 변경 이력을 기록할 수 있는 최소 기능을 구축한다.

이 단계의 핵심은 화려한 기능이 아니라  
**식료품 상태 데이터와 이력 데이터가 안정적으로 관리되는 구조를 만드는 것**이다.

---

## 목적
- 사용자별로 데이터를 분리해 관리할 수 있어야 한다
- 현재 보유 중인 식료품 상태를 저장하고 조회할 수 있어야 한다
- 입고 / 사용 / 폐기 / 수정 이력을 남길 수 있어야 한다
- 최소한의 편의기능으로 기본 입력 부담을 줄여야 한다

---

## 핵심 검증 포인트
- 사용자가 식료품을 직접 등록하고 관리할 수 있는가
- 현재 상태와 이력 구조가 자연스럽게 이어지는가
- 최소 기능만으로도 서비스의 기본 흐름이 성립하는가

---

## 포함 기능

### 1. 회원 관리
- 회원 가입
- 로그인 / 로그아웃
- 사용자별 데이터 분리

### 2. 식료품 종류 관리
- 식료품 종류 등록 / 수정 / 삭제
- 이름 관리
- 기본 보관 장소 설정
- 평균 유통기한 기본값 설정

### 3. 식료품 보관 상태 관리
- 현재 보유 식료품 등록 / 수정 / 삭제
- 수량 / 단위 입력
- 유통기한 입력
- 보관 장소 입력
- 현재 보유 목록 조회

기본 보관 장소 예시:
- 냉장고
- 냉동고

### 4. 식료품 이력 관리
- 입고 이력
- 사용 이력
- 폐기 이력
- 수정 이력
- 항목별 / 전체 이력 조회

### 5. 기본 편의기능
- 평균 유통기한 자동 입력
- 기본 보관 장소 자동 선택
- 유통기한 임박 표시
- 최근 등록 항목 간단 재사용

---

## 데이터 구조 관점
이 단계에서는 아래 구조를 명확히 가져간다.

- **식료품 종류**
- **식료품 보관 상태**
- **식료품 변경 이력**

즉:
- 현재 상태는 별도 관리
- 상태 변화는 이력으로 누적

이 구조가 이후 UX 개선과 추천 기능의 기반이 된다.

---

## 제외 범위
1차에서는 아래 기능은 포함하지 않는다.

- OCR 기반 영수증 등록
- 복잡한 UX 최적화
- 레시피 추천
- 장보기 추천
- 식단 / 플랜 기능

---

## 1차 완료 기준
- 회원이 로그인 후 자신의 식료품을 관리할 수 있다
- 식료품 현재 상태를 조회할 수 있다
- 변경 시 이력이 자동 기록된다
- 유통기한 임박 항목을 확인할 수 있다

---
