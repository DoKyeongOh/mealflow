# 식료품 관리 서비스 아이디어 요약

## 1. 아이디어 개요
집에 보관 중인 식료품을 관리하는 서비스다.  
핵심은 **식료품의 현재 상태 관리**와 **입출력 이력 관리**이며,  
여기에 **레시피 추천**, **유통기한 알림**, **장보기 보조** 같은 결정 지원 기능을 결합하는 방향이다.

단순 재고 관리보다,  
**집에 있는 재료로 무엇을 먹을지 빠르게 정하게 해주는 서비스**를 지향한다.

---

## 2. 핵심 문제의식
식료품 관리 서비스의 가장 큰 문제는 기능 부족이 아니라 **유저 입력 부담**이다.

사용자는 보통:
- 냉장고에 뭐가 있는지 정리하고 싶어함
- 유통기한을 놓치고 싶지 않아함
- 중복 구매를 줄이고 싶어함

하지만 실제로는:
- 식료품을 매번 수동 등록하기 귀찮아함
- 사용 / 폐기 / 수정까지 계속 기록하지 않음
- 관리 자체가 습관이 되지 않으면 서비스 사용이 끊김

따라서 이 서비스의 핵심 성공 조건은  
**입력 마찰을 얼마나 줄이느냐** 와  
**기록한 데이터가 바로 행동 가치로 연결되느냐** 이다.

---

## 3. 기본 기능 구조

### 3-1. 식료품 종류 관리
식료품의 기준 정보를 관리한다.

예시 속성:
- 이름
- 기본 보관 타입
- 평균 유통기한 기본값

### 3-2. 식료품 보관 상태 관리
현재 집에 어떤 식료품이 어떻게 보관되고 있는지 저장한다.  
이 테이블은 상태성 데이터다.

예시 속성:
- 식료품 종류 ID
- 수량
- 단위
- 유통기한
- 보관 장소
- 상태

보관 장소는 사용자 입력값을 받되,
기본값으로는 아래를 제공한다.
- 냉장고
- 냉동고

### 3-3. 입출력 / 변경 이력 관리
식료품의 변화 기록을 누적한다.

예시 이벤트:
- 입고
- 사용
- 폐기
- 수정

즉, 구조는 다음과 같다:
- **현재 상태 테이블**
- **이벤트 로그 테이블**

---

## 4. 입력 부담을 줄이기 위한 방향

### 4-1. 평균 유통기한 자동 입력
식료품 종류별 평균 유통기한을 디폴트로 지정해,
등록 시 자동으로 채워준다.

의미:
- 사용자가 매번 직접 입력하지 않아도 됨
- 정확한 정답이 아니라 빠른 초안 제공 역할

### 4-2. 영수증 촬영 기반 일괄 등록
영수증 이미지를 인식해 식료품 후보를 추출하고,
여러 항목을 한 번에 등록할 수 있도록 한다.

초기 방향:
- 완전 자동 등록보다
- **인식 결과 후보를 보여주고 사용자가 선택 후 등록**하는 방식이 현실적임

### 4-3. 이전 입력 형식 재사용
사용자가 예전에 입력했던 방식과 값을 기억해 재사용한다.

예:
- 자주 입력하는 품목
- 최근 등록 항목 재추가
- 같은 품목의 기본 보관 방식 / 기본 수량 제안

### 4-4. 쉬운 선택형 UI
복잡한 입력폼보다 빠른 선택과 최소 터치 중심 UI를 제공한다.

예:
- 보관 장소 원터치 선택
- 수량 증감 버튼
- 최근 항목 빠른 등록
- 임박 / 사용완료 상태 빠른 변경

핵심은 드래그 자체보다  
**조작 횟수 최소화**다.

---

## 5. 사용 이유가 되는 핵심 부가 기능

### 5-1. 자동 레시피 추천
보유 중인 식료품을 기준으로 만들 수 있는 레시피를 추천한다.

추천 방향:
- 지금 있는 재료로 가능한 레시피
- 1~2개만 더 사면 가능한 레시피
- 유통기한 임박 재료 우선 소진 레시피
- 조리 시간 / 난이도 기준 필터링

이 기능은 단순 관리 툴을 넘어  
**식사 결정 보조 서비스**로 확장시키는 핵심이다.

### 5-2. 유통기한 알리미
유통기한 임박 재료를 알려준다.

좋은 방향:
- 오늘 먹어야 할 재료
- 이번 주 안에 처리할 재료
- 버릴 가능성이 높은 재료 표시

즉, 단순 알림보다  
**행동 유도형 알림**이 더 중요하다.

### 5-3. 장보기 도우미
레시피는 있는데 부족한 재료가 있을 경우,
장볼 품목을 추천한다.

예:
- 만들고 싶은 레시피 선택
- 없는 재료 자동 추출
- 장보기 리스트로 전환

### 5-4. 상황별 음식 플랜
특정 기간 동안 사용자가 지정한 레시피를 소비할 수 있도록
식사 플랜을 제공한다.

예:
- 이번 주 식단 플랜
- 임박 재료 소진 중심 플랜
- 혼밥 / 다이어트 / 간단식 중심 플랜

이 기능은 확장성은 높지만,
초기 MVP보다는 이후 단계 기능에 가깝다.

## 6. 이 서비스의 핵심 가치 정의

### 약한 정의
- 식료품을 기록하고 관리하는 서비스

이 정의만으로는 사용 지속성이 약할 가능성이 크다.

### 더 강한 정의
- **집에 있는 재료로 무엇을 먹을지 바로 정하게 해주는 서비스**
- **버리기 전에 먹게 도와주는 서비스**
- **생각하기 귀찮을 때 식사 결정을 줄여주는 서비스**

즉,  
이 서비스는 단순 재고 관리보다  
**결정 보조와 소비 유도**에 더 큰 가치가 있다.

---

## 7. MVP 우선순위 제안

### 1차 MVP
- 식료품 종류 관리
- 식료품 보관 상태 관리
- 평균 유통기한 자동 입력
- 최근 입력 재사용
- 유통기한 임박 표시
- 보유 재료 기반 간단 레시피 추천

### 2차
- 영수증 인식 후보 등록
- 부족 재료 장보기 추천
- 알림 고도화

### 3차
- 음식 플랜 제공
- 개인 취향 / 목적 반영
- 소비 우선순위 최적화

---

## 8. 최종 정리
이 아이디어의 본질은 단순한 식료품 CRUD가 아니다.

- **식료품 관리**는 진입 구조
- **레시피 추천 / 유통기한 알림 / 장보기 추천**은 사용 지속 이유
- **입력 자동화**는 서비스 성패를 좌우하는 핵심 조건

따라서 이 프로젝트는  
“냉장고 재고를 기록하는 앱”보다는  
**집에 있는 재료를 바탕으로 먹는 결정을 돕는 서비스**로 정의하는 것이 더 적절하다.
