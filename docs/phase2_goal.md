# 2차 목표 정리

## 목표
1차에서 구축한 기본 구조를 바탕으로,  
식료품의 입출력 과정을 단순화하고 사용자가 덜 귀찮게 기록할 수 있도록 UX를 개선한다.

이 단계의 핵심은 기능 추가 자체보다  
**입력 마찰을 줄여 실제 사용성을 높이는 것**이다.

---

## 목적
- 반복 입력을 줄인다
- 등록 / 사용 / 폐기 과정을 더 빠르게 만든다
- 모바일이나 실생활 흐름에서 쉽게 쓸 수 있도록 한다
- 사용자가 기록을 지속할 수 있도록 UX를 개선한다

---

## 핵심 검증 포인트
- 사용자가 덜 귀찮게 입력할 수 있는가
- 반복 작업이 충분히 줄어드는가
- 입력 시간이 체감될 정도로 짧아지는가
- 상태 변경이 빠르고 직관적인가

---

## 포함 기능

### 1. 반복 입력 최소화
- 이전 입력값 자동완성
- 최근 등록 항목 빠른 재등록
- 자주 쓰는 식료품 템플릿 제공
- 품목별 기본 수량 / 단위 / 보관 장소 기억

### 2. 입출력 단순화
- 사용 처리 버튼
- 폐기 처리 버튼
- 수량 +1 / -1 조정
- 상세 수정 없이 빠른 상태 변경

### 3. UX 개선
- 빠른 등록 흐름 개선
- 홈 화면에서 현재 보유 식료품 중심으로 확인 가능
- 유통기한 임박 항목 강조
- 선택 중심 UI 제공
- 최소 터치로 처리 가능한 화면 구성

### 4. 영수증 기반 입력 보조
- 영수증 이미지 업로드
- OCR 또는 품목 후보 추출
- 사용자 확인 후 일괄 등록

초기 방향:
- 완전 자동 등록이 아니라
- **후보 제안 후 선택 등록 방식**으로 시작

---

## UX 관점 핵심 원칙
이 단계에서는 드래그 같은 특정 인터랙션보다  
아래 원칙이 더 중요하다.

- 터치 수 최소화
- 반복 입력 최소화
- 빠른 상태 변경
- 현재 필요한 정보 우선 노출

즉,  
**예쁜 UI보다 빠른 사용 흐름**이 우선이다.

---

## 제외 범위
2차에서는 아래 기능은 본격적으로 포함하지 않는다.

- 고도화된 개인화 추천
- 복잡한 레시피 추천 엔진
- 장보기 최적화
- 주간 / 기간별 식단 플랜

---

## 2차 완료 기준
- 반복 입력이 줄어들었다
- 등록 / 사용 / 폐기 처리 시간이 짧아졌다
- 빠른 재등록과 빠른 상태 변경이 가능하다
- 영수증 기반 보조 등록이 동작한다
- 실사용성 측면에서 1차보다 명확히 편해졌다

---

**입력 부담을 줄이고 실제 생활에서 계속 쓸 수 있게 만드는 사용성 개선 단계**
