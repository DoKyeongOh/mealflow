# 3차 목표 정리

## 목표
기록된 식료품 데이터를 레시피와 연결해,  
사용자가 보유 재료를 바탕으로 메뉴를 추천받고 식사 계획까지 세울 수 있도록 확장한다.

이 단계의 핵심은 단순 재고 관리 앱을 넘어서  
**먹는 결정을 돕는 서비스로 발전시키는 것**이다.

---

## 목적
- 보유 재료를 기반으로 메뉴를 추천한다
- 유통기한 임박 재료를 소비로 연결한다
- 필요한 경우 장볼 재료를 제안한다
- 일정 기간의 식사 계획까지 확장한다

---

## 핵심 검증 포인트
- 보유 재료를 활용한 메뉴 추천이 실제로 유용한가
- 추천이 소비 행동으로 이어지는가
- 유통기한 임박 재료를 자연스럽게 소진하게 만드는가
- 재고 데이터가 식사 계획 수립에 활용될 수 있는가

---

## 포함 기능

### 1. 레시피 연동
- 레시피 데이터 연동 또는 자체 레시피 DB 구성
- 레시피별 필요 재료 관리
- 보유 재료와 레시피 재료 매칭

### 2. 자동 메뉴 추천
- 지금 있는 재료로 가능한 메뉴 추천
- 부족한 재료가 적은 메뉴 우선 추천
- 유통기한 임박 재료 우선 사용 메뉴 추천
- 조리 시간 / 난이도 / 상황별 필터

### 3. 장보기 도우미
- 원하는 레시피 기준 부족 재료 추출
- 장보기 리스트 생성
- 현재 보유 재료와 비교해 필요한 항목만 추천

### 4. 플랜 관리
- 특정 기간 식사 계획 생성
- 임박 재료 소진 중심 플랜 제공
- 사용자가 고른 레시피 기반 주간 플랜
- 플랜에 맞춘 재료 소비 순서 안내

---

## 추천 기능의 초기 원칙
추천 기능은 처음부터 과하게 복잡하게 시작하지 않는다.

우선순위:
1. 재료 일치 기반 추천
2. 부족 재료 개수 기준 정렬
3. 유통기한 임박 재료 가중치 반영
4. 이후 개인 취향 / 선호 반영

즉,  
처음엔 **단순하지만 납득 가능한 추천**이 더 중요하다.

---

## 서비스 관점 의미
이 단계부터 서비스의 정의가 달라진다.

기존:
- 식료품을 기록하는 서비스

확장 후:
- 집에 있는 재료로 무엇을 먹을지 정하게 해주는 서비스
- 버리기 전에 먹도록 유도하는 서비스
- 장보기와 식사 계획까지 연결하는 서비스

---

## 제외 범위
3차에서도 처음부터 아래까지 한 번에 다 하진 않는다.

- 지나치게 복잡한 개인 취향 모델
- 영양 최적화 수준의 식단 엔진
- 과도하게 자동화된 풀 오토 플랜 생성

초기에는  
**재고 기반 추천 → 부족 재료 제안 → 간단한 플랜** 순으로 확장한다.

---

## 3차 완료 기준
- 보유 재료 기반 메뉴 추천이 가능하다
- 부족 재료를 장보기 리스트로 전환할 수 있다
- 유통기한 임박 재료 활용 추천이 가능하다
- 일정 기간 식사 플랜을 구성할 수 있다

---
