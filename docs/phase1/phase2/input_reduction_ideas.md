# 사용자 입출력 감소 아이디어 정리

## 1. 목적
식료품 관리 서비스의 가장 큰 리스크는 사용자가 식료품을 계속 직접 입력하고 수정해야 한다는 점이다.

따라서 이 문서는 사용자의 입력 부담을 줄이고,
식료품 관리 행동이 자연스럽게 이어지도록 만드는 기능 아이디어를 정리한다.

핵심 방향은 다음과 같다.

- 등록 시 입력값을 줄인다
- 사용 / 폐기 / 이동 처리를 단순화한다
- 사용자가 실제 생활 흐름 속에서 자연스럽게 앱을 쓰게 만든다
- 레시피 기능과 연결될 수 있는 확장 포인트를 남긴다

---

## 2. 바로 적용하기 좋은 아이디어

### 2.1 최근 등록 재사용
사용자가 최근 등록했던 식료품 정보를 다시 활용한다.

#### 예시
- 계란 30개 / 냉장 / 21일
- 우유 1L / 냉장 / 7일
- 양파 3개 / 실온 / 14일

다음 등록 시 최근 등록 항목을 선택하면,
기존 입력값이 자동으로 채워진다.

#### 기대 효과
- 반복 입력 감소
- 자주 사는 품목 등록 속도 향상
- 사용자가 식료품 종류를 매번 다시 입력하지 않아도 됨

#### 적용 우선순위
높음

---

### 2.2 영수증 OCR 후보 등록
영수증 이미지를 업로드하면,
OCR 또는 텍스트 추출을 통해 식료품 후보를 제안한다.

#### 권장 흐름
1. 영수증 이미지 업로드
2. 품목 후보 추출
3. 식료품으로 등록할 항목 선택
4. 수량 / 보관장소 / 유통기한 기본값 자동 제안
5. 사용자 확인 후 일괄 등록

#### 주의점
완전 자동 등록은 위험하다.

영수증에는 아래 문제가 있을 수 있다.

- 품목명이 축약되어 있음
- 식료품이 아닌 항목이 섞여 있음
- 수량과 단위가 불명확할 수 있음
- OCR 결과가 틀릴 수 있음

따라서 초기에는 **후보 제안 후 사용자 선택 등록** 방식이 적절하다.

#### 기대 효과
- 여러 품목을 한 번에 등록 가능
- 장보기 후 입력 부담 감소
- 대량 등록 경험 개선

#### 적용 우선순위
중간 이상

---

### 2.3 빠른 사용 / 폐기 버튼
식료품 상세 화면에 들어가지 않고,
목록에서 바로 사용 또는 폐기 처리할 수 있게 한다.

#### 예시 액션
- 1개 사용
- 절반 사용
- 전부 사용
- 폐기
- 수량 직접 입력 후 사용

#### 기대 효과
- 소비 기록 부담 감소
- 재고 수정 흐름 단축
- 사용자가 기록을 미루지 않게 됨

#### 적용 우선순위
높음

---

### 2.4 홈 화면을 입력 중심으로 구성
홈 화면을 단순 조회 화면이 아니라,
등록 / 사용 / 폐기 처리가 바로 가능한 작업 화면으로 구성한다.

#### 추천 홈 구성
- 유통기한 임박 식료품
- 최근 등록 항목
- 자주 쓰는 항목
- 보관 장소별 현재 보유 목록
- 빠른 추가 버튼
- 빠른 사용 / 폐기 버튼

#### 기대 효과
- 사용자가 메뉴를 찾아 들어가지 않아도 됨
- 앱을 열자마자 필요한 행동을 할 수 있음
- 실사용성이 높아짐

#### 적용 우선순위
높음

---

### 2.5 보관 장소 탭
보관 장소 기준으로 식료품을 나누어 보여준다.

#### 기본 탭 예시
- 냉장고
- 냉동고
- 실온
- 기타

#### UX 아이디어
- 냉장고 탭에서 추가하면 기본 보관 장소가 냉장고로 설정
- 냉동고 탭에서 추가하면 기본 보관 장소가 냉동고로 설정
- 탭별 임박 항목 개수 표시

#### 기대 효과
- 조회 속도 향상
- 등록 시 보관 장소 입력 감소
- 사용자의 실제 냉장고 정리 흐름과 맞음

#### 적용 우선순위
높음

---

## 3. 추가로 추천하는 아이디어

### 3.1 임박 재료 기반 빠른 액션
유통기한 임박 목록에서 바로 처리 액션을 제공한다.

#### 예시 액션
- 오늘 먹음
- 냉동으로 이동
- 전부 사용
- 폐기
- 유통기한 수정

#### 기대 효과
- 유통기한 알림이 실제 행동으로 이어짐
- 버려지는 식료품 감소
- 임박 식료품을 따로 찾아 수정할 필요가 없음

#### 적용 우선순위
높음

---

### 3.2 같은 조건으로 추가
재고 목록이나 이력 화면에서,
기존 항목과 같은 조건으로 식료품을 다시 등록할 수 있게 한다.

#### 예시
지난번 등록:
- 계란 30개
- 냉장
- 평균 유통기한 21일

`같은 조건으로 추가`를 누르면:
- 수량, 단위, 보관 장소는 그대로 적용
- 유통기한은 오늘 기준으로 다시 계산

#### 기대 효과
- 최근 등록 재사용보다 더 직접적인 반복 등록 지원
- 자주 사는 품목의 등록 속도 향상

#### 적용 우선순위
높음

---

### 3.3 품목 검색 + 없으면 즉시 생성
식료품 종류 관리 화면을 따로 거치지 않고,
보관 항목 등록 흐름 안에서 식료품 종류를 바로 만들 수 있게 한다.

#### 권장 흐름
1. 사용자가 품목명 입력
2. 기존 `GroceryType` 검색
3. 있으면 선택
4. 없으면 즉시 생성
5. 곧바로 `InventoryItem` 등록

#### 기대 효과
- 식료품 종류 관리와 보관 항목 등록이 분리되어 생기는 불편 감소
- 첫 사용 경험 개선
- 사용자가 CRUD 구조를 의식하지 않아도 됨

#### 적용 우선순위
매우 높음

---

### 3.4 기본 단위 자동 추천
품목별로 자주 쓰는 단위를 기억하거나 기본 단위를 제안한다.

#### 예시
- 계란 → 개
- 우유 → ml 또는 L
- 쌀 → kg
- 양파 → 개
- 닭가슴살 → 팩

#### 기대 효과
- 단위 선택 부담 감소
- 입력 일관성 향상
- 수량 입력 속도 개선

#### 적용 우선순위
중간 이상

---

### 3.5 대충 입력 모드
정확한 수량이나 유통기한을 모르는 경우에도 식료품을 등록할 수 있게 한다.

#### 예시 수량
- 조금
- 보통
- 많이
- 반 정도
- 한 봉지

#### 예시 데이터 구조
- `quantity`
- `unit`
- `quantity_precision`

`quantity_precision` 예시:
- `EXACT`
- `APPROXIMATE`

#### 기대 효과
- 정확한 정보를 모른다는 이유로 등록을 포기하지 않게 함
- 생활형 재고 관리에 더 잘 맞음
- 입력 부담 감소

#### 적용 우선순위
중간 이상

---

### 3.6 유통기한 모름 옵션
유통기한을 모르는 식료품도 등록할 수 있게 한다.

#### 선택지 예시
- 평균 유통기한으로 자동 설정
- 유통기한 모름
- 나중에 입력
- 임박 관리 제외

#### 기대 효과
- 유통기한 입력이 등록의 장애물이 되지 않음
- 정확하지 않은 값 입력을 강요하지 않음
- 사용자가 일단 재고를 등록하게 만들 수 있음

#### 적용 우선순위
높음

---

### 3.7 자주 쓰는 필터 저장
조회 시 자주 쓰는 필터 조합을 저장한다.

#### 예시
- 냉장고 + 임박순
- 냉동고 + 오래된순
- 실온 + 보유중
- 이번 주 처리할 것

#### 기대 효과
- 조회 조건을 매번 설정하지 않아도 됨
- 보관 장소 / 상태 / 유통기한 기준 조회가 빨라짐

#### 적용 우선순위
중간

---

### 3.8 일괄 선택 후 처리
여러 식료품을 선택해 한 번에 처리할 수 있게 한다.

#### 예시 액션
- 냉동고로 이동
- 폐기 처리
- 보관 장소 변경
- 사용 완료
- 유통기한 수정

#### 기대 효과
- 냉장고 정리 시 편리함
- 여러 항목을 하나씩 수정하는 부담 감소
- 정리 모드와 함께 사용하기 좋음

#### 적용 우선순위
중간 이상

---

### 3.9 냉장고 정리 모드
사용자가 냉장고를 정리하는 상황에 맞춘 별도 모드를 제공한다.

#### 권장 흐름
1. 냉장고 정리 시작
2. 보관 장소별 목록 표시
3. 실제 남아있는 항목만 체크
4. 없는 항목은 사용 또는 폐기 후보로 처리
5. 새로 발견한 항목은 빠르게 추가

#### 기대 효과
- 앱 사용 시점을 실제 생활 흐름과 연결
- 재고 데이터 최신화 유도
- 단순 CRUD보다 더 명확한 사용 이유 제공

#### 적용 우선순위
매우 높음

---

### 3.10 알림에서 바로 액션
유통기한 알림이나 홈 카드에서 바로 상태 변경이 가능하게 한다.

#### 예시 액션
- 먹었음
- 냉동함
- 나중에 보기
- 폐기함

#### 기대 효과
- 알림이 단순 정보 전달에서 행동 유도로 바뀜
- 앱 진입 후 탐색 비용 감소
- 임박 식료품 처리율 증가

#### 적용 우선순위
중간 이상

---

## 4. 레시피 기능 추가 후 좋아지는 아이디어

### 4.1 자주 사는 품목 템플릿
레시피나 식사 패턴이 쌓이면,
자주 사는 품목 조합을 템플릿으로 제공할 수 있다.

#### 예시
- 아침 기본 세트
- 주말 장보기 세트
- 찌개 재료 세트
- 샐러드 재료 세트

#### 기대 효과
- 반복 장보기 등록 감소
- 식사 패턴과 재고 관리 연결

---

### 4.2 장보기 묶음 등록
레시피 기반으로 필요한 재료 묶음을 한 번에 등록하거나 장보기 리스트로 만들 수 있다.

#### 예시
김치찌개 재료 묶음:
- 김치
- 돼지고기
- 두부
- 대파

#### 기대 효과
- 레시피와 장보기 흐름 연결
- 사용자가 개별 품목을 하나씩 입력하지 않아도 됨

---

### 4.3 먹은 메뉴 기준 자동 차감 후보
사용자가 특정 메뉴를 먹었다고 선택하면,
해당 레시피에 포함된 재료를 자동 차감 후보로 제안한다.

#### 권장 흐름
1. 사용자가 `김치찌개 먹음` 선택
2. 사용된 재료 후보 표시
3. 사용자가 실제 사용한 재료만 체크
4. 선택한 항목 차감

#### 주의점
완전 자동 차감은 위험하다.

실제 사용량은 레시피와 다를 수 있으므로,
초기에는 **자동 차감 후보 제안 + 사용자 확인** 방식이 적절하다.

#### 기대 효과
- 사용 이력 기록 부담 감소
- 레시피 기능과 재고 관리가 자연스럽게 연결됨

---

### 4.4 이 재료 써먹기
식료품 상세 화면에서 해당 재료를 활용할 수 있는 레시피를 바로 추천한다.

#### 예시
- 두부 상세 화면
- `두부 써먹기` 버튼
- 두부조림, 된장찌개, 마파두부 추천

#### 기대 효과
- 재고 조회가 메뉴 결정으로 연결됨
- 임박 재료 소비를 유도하기 좋음

---

### 4.5 임박 재료 우선 레시피
유통기한이 임박한 재료를 우선 사용하도록 레시피를 추천한다.

#### 예시
임박 재료:
- 두부
- 애호박

추천 메뉴:
- 된장찌개
- 두부조림
- 애호박전

#### 기대 효과
- 유통기한 알림의 실질적 가치 증가
- 버리는 식재료 감소
- 서비스 사용 이유가 강해짐

---

---

---

## 6. 최종 추천
가장 먼저 잡아야 할 것은 OCR 같은 자동화보다,
사용자가 매일 쓰는 흐름을 줄이는 기능이다.

특히 아래 3개는 강하게 추천한다.

### 1순위: 품목 검색 + 없으면 즉시 생성
식료품 종류 관리와 보관 항목 등록을 분리하면 초반 UX가 답답해진다.  
등록 흐름 안에서 종류 생성까지 끝나야 한다.

### 2순위: 냉장고 정리 모드
사용자가 앱을 쓰는 현실적인 순간을 만들어준다.  
단순히 기록하라고 요구하는 것보다, 정리하는 행동에 맞춰 앱을 쓰게 하는 편이 자연스럽다.

### 3순위: 대충 입력 모드
정확한 수량과 유통기한을 강요하면 사용자가 입력을 포기할 수 있다.  
생활형 관리 서비스에서는 대략적인 입력을 허용하는 것이 중요하다.

## 7. 한 줄 정리
입출력 감소의 핵심은 자동화 자체보다,
**기본값, 재사용, 원터치 처리, 실제 생활 흐름에 맞춘 UX** 를 만드는 것이다.
