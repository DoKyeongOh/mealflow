# [Feature] 초기 사용성 개선을 위한 시스템 프리셋 (System Presets)

## 1. 개요
현재 Mealflow는 사용자가 직접 모든 저장 장소와 식료품 마스터 정보를 입력해야 하는 구조입니다. 이는 초기 사용자가 서비스를 시작할 때 큰 진입장벽이 되며, 반복적인 입력 부담을 가중시킵니다. 이를 해결하기 위해 시스템에서 미리 정의된 프리셋(Preset) 정보를 제공하여, 선택 중심의 UI를 통해 입력 마찰을 최소화하고자 합니다.

## 2. 목적
- 초기 가입 사용자의 셋업 과정 단축
- 보편적으로 사용하는 저장 장소 및 식료품 정보의 빠른 등록 지원
- "입력" 중심이 아닌 "선택" 중심의 사용자 경험 제공

## 3. 상세 요구사항

### 3.1 저장 장소 프리셋 (StorageLocationPreset)
사용자가 자신만의 저장 장소를 만들기 전, 시스템이 제안하는 기본 목록을 제공합니다.

- **기능**:
    - 시스템은 보편적인 저장 장소 목록(냉장고, 냉동고, 실온, 팬트리 등)을 제공합니다.
    - 저장 장소 관리 페이지에서 프리셋 목록을 노출합니다.
    - 사용자가 프리셋 항목을 선택하면, 해당 이름으로 사용자의 `StorageLocation`이 즉시 생성됩니다.
    - 생성된 후에는 사용자가 이름을 수정하거나 삭제할 수 있는 일반 `StorageLocation`과 동일하게 동작합니다.
- **데이터 예시**:
    - 냉장고, 냉동고, 실온, 팬트리, 김치냉장고 등

### 3.2 식료품 마스터 프리셋 (GroceryPreset)
식료품을 등록(InventoryItem 생성)할 때, 사용자가 직접 정의한 식료품 종류(`GroceryType`)뿐만 아니라 시스템이 제공하는 프리셋에서도 검색하고 선택할 수 있게 합니다.

- **기능**:
    - 보관 항목 등록 시 식료품 검색창에서 사용자의 `GroceryType`과 시스템의 `GroceryPreset`을 통합 검색합니다.
    - 사용자가 `GroceryPreset` 항목을 선택하여 등록할 경우:
        1. 해당 프리셋 정보를 바탕으로 사용자의 `GroceryType`을 자동 생성합니다.
        2. 생성된 `GroceryType`을 참조하여 `InventoryItem`을 저장합니다.
    - 이를 통해 사용자는 매번 식료품 종류를 먼저 만드는 번거로움 없이 검색-선택만으로 등록을 완료할 수 있습니다.
- **데이터 예시**:
    - 계란 (유통기한 기본값: 21일, 기본 보관장소: 냉장)
    - 우유 (유통기한 기본값: 7일, 기본 보관장소: 냉장)
    - 양파 (유통기한 기본값: 14일, 기본 보관장소: 실온)

## 4. 데이터 모델 설계 (안)

### 4.1 StorageLocationPreset (저장 장소 프리셋)
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| id | Long | PK |
| name | String | 저장 장소 명칭 (ex: 냉장고) |
| icon_url | String | (선택) 표시할 아이콘 경로 |
| display_order | Integer | 노출 순서 |

### 4.2 GroceryPreset (식료품 프리셋)
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| id | Long | PK |
| name | String | 식료품 명칭 (ex: 계란) |
| default_shelf_life_days | Integer | 기본 권장 유통기한 |
| default_storage_preset_id | Long | 기본 권장 저장 장소 프리셋 ID |

## 5. 변경되는 프로세스

### 5.1 저장 장소 생성 흐름
1. 저장 장소 목록 페이지 진입
2. 상단 또는 팝업에 "추천 저장 장소(Preset)" 노출
3. 사용자가 "냉장고" 선택
4. 백엔드 호출: `POST /api/storage-locations/from-preset` (presetId 전달)
5. 사용자의 `StorageLocation` 테이블에 "냉장고" 레코드 생성

### 5.2 식료품 등록 흐름 (Input Reduction 적용)
1. 보관 항목 등록 화면 진입
2. 식료품 이름 입력 (Search)
3. API 호출: 사용자의 `GroceryType` + 시스템의 `GroceryPreset` 통합 검색 결과 반환
4. 사용자가 프리셋 항목 "우유" 선택
5. UI에서 "우유"의 기본 유통기한(7일)과 기본 보관장소를 자동 세팅
6. 등록 완료 시:
    - (필요 시) `GroceryType` 자동 생성 후 `InventoryItem` 저장
    - 또는 `InventoryItem` 저장 시 프리셋 ID를 넘겨 백엔드에서 처리

## 6. 기대 효과
- **UX 개선**: 복잡한 입력 없이 몇 번의 클릭/터치만으로 초기 데이터 구성 완료
- **데이터 정형화**: 사용자들이 보편적인 명칭을 사용하게 유도하여 데이터 일관성 확보
- **Phase 2 목표 달성**: 입력 마찰을 줄여 실제 생활에서의 사용성을 대폭 향상
