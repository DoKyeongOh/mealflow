# [Feature] 저장 장소 프리셋 (Storage Location Presets)

## 1. 개요
사용자가 저장 장소를 매번 직접 입력하는 번거로움을 줄이기 위해, 시스템에서 정의한 기본 저장 장소 목록(프리셋)을 제공하고 이를 활용해 빠르게 장소를 생성할 수 있도록 합니다.

## 2. 목적
- 초기 사용자의 저장 장소 설정 편의성 증대
- 태그 형태의 UI를 통한 퀵 버튼 제공
- 프리셋 외에 사용자가 직접 이름을 지정하여 등록할 수 있는 유연함 제공

## 3. 상세 요구사항

### 3.1 UI/UX 구성
- **저장 장소 목록 화면**:
    - 화면 하단 또는 특정 영역에 프리셋 저장 장소(냉장고, 냉동고, 실온 등)를 **태그(Tag)** 형식으로 나열합니다.
    - 나열된 프리셋 태그를 클릭하면 해당 명칭으로 사용자의 저장 장소가 즉시 등록됩니다.
    - 프리셋 태그 목록 최우측에는 **[직접 등록]** 버튼을 배치합니다.
- **직접 등록 모달**:
    - [직접 등록] 버튼 클릭 시 "저장 장소 등록하기" 모달창이 팝업됩니다.
    - 모달 구성 요소:
        - **Select Box**: 전체 프리셋 목록 중 하나를 선택할 수 있는 선택창
        - **입력창**: 선택한 프리셋 외에 사용자가 원하는 커스텀 이름을 입력할 수 있는 필드
        - **등록 버튼**: 최종적으로 입력된 이름으로 저장 장소를 생성합니다.

### 3.2 데이터 모델 설계 (안)
#### StorageLocationPreset (저장 장소 프리셋)
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| id | String | 프리셋 코드 (ex: FRIDGE, FREEZER) |
| name | String | 사용자에게 보여질 명칭 (ex: 냉장고, 냉동고) |

## 4. 프로세스
1. 사용자가 저장 장소 관리 페이지 진입
2. 프리셋 태그(예: 냉장고) 클릭 시:
    - `POST /api/storage-locations/from-preset` 호출 (presetId: 'FRIDGE')
    - 사용자 소유의 '냉장고' 저장 장소 생성
3. [직접 등록] 클릭 시:
    - 모달 오픈 -> Select Box에서 프리셋 선택 또는 커스텀 이름 입력
    - 등록 버튼 클릭 시 사용자가 지정한 이름으로 저장 장소 생성
