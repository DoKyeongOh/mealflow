# [Feature] 식료품 마스터 프리셋 (Grocery Presets)

## 1. 개요
식료품 보관 항목을 등록할 때 사용자가 직접 식료품 마스터 정보를 매번 생성할 필요 없이, 시스템이 제공하는 방대한 식료품 프리셋 데이터를 검색하여 바로 활용할 수 있도록 합니다.

## 2. 목적
- 식료품 등록 과정에서의 입력 최소화 (Input Reduction)
- 보편적인 식료품의 권장 유통기한 및 보관 장소 가이드 제공
- 사용자의 개인 식료품 마스터(`GroceryType`)와 시스템 프리셋의 유연한 통합

## 3. 상세 요구사항

### 3.1 검색 및 선택 흐름
- **통합 검색**: 보관 항목 등록 시 식료품 이름을 입력하면, 사용자가 이미 등록한 `GroceryType` 목록과 시스템이 제공하는 `GroceryPreset` 목록을 동시에 검색하여 결과를 보여줍니다.
- **프리셋 선택 시 동작**:
    - 사용자가 `GroceryPreset` 항목(예: "계란")을 선택하면, 해당 프리셋에 정의된 `default_shelf_life_days`(21일)와 `default_storage_preset_id`(냉장) 정보가 UI에 자동으로 채워집니다.
    - 사용자는 자동 완성된 정보를 확인하거나 수정한 후 등록을 완료합니다.
- **백엔드 처리**:
    - 프리셋을 이용해 등록할 경우, 백엔드에서는 해당 정보를 바탕으로 사용자의 `GroceryType` 레코드를 자동으로 생성하거나, 기존에 동일한 이름의 `GroceryType`이 있다면 이를 연결합니다.

### 3.2 데이터 모델 설계 (안)
#### GroceryPreset (식료품 프리셋)
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| id | String | 프리셋 코드 (ex: EGG, MILK) |
| name | String | 식료품 명칭 (ex: 계란, 우유) |
| default_shelf_life_days | Integer | 기본 권장 유통기한 (일 단위) |
| default_storage_preset_id | String | 기본 권장 저장 장소 프리셋 ID |

## 4. 기대 효과
- **사용성 대폭 개선**: 검색과 클릭만으로 복잡한 식료품 정보 입력 완료
- **데이터 일관성**: 시스템 표준 명칭 사용 유도로 추후 레시피 추천 등과의 연계 용이성 확보
